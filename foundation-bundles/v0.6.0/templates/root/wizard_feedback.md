# Wizard Feedback

*This file captures issues in the running system that trace back to the wizard — a question not asked, a setting configured with insufficient information, or something under-explained during setup. It is the bridge from system experience back to wizard improvement.*

*When an agent encounters an issue that is not about system tuning but about how the wizard built the system, append an entry below.*

---

## How to write entries

**Three fields per entry. Nothing else.**

- **Step:** Which wizard interview step the issue relates to (00–15, or "manual", "about")
- **What happened:** Plain-language description of what the system encountered. Use descriptions of wizard behavior only — no file paths, URLs, credentials, agent names, or technical identifiers.
- **What expected:** What the user thought should happen

**Before sharing this file:** Review it and delete anything that names your project, your company, or a specific person.
