# Architectural Review Staging

*Staging file for architectural review findings categorized as "note for phase-gate." Findings written here are automatically incorporated into the next phase-gate review. This file is never surfaced to the user as a separate action item — it surfaces as part of the gate review.*

*Updated by the system. Never edited manually.*

---

## Staged Findings

*No staged findings.*

---

## When populated, each entry contains:

| Field | Contents |
|-------|---------|
| Finding ID | Unique ID for this finding (e.g., ARF-001) |
| Date raised | When the finding was identified |
| Trigger | What event triggered the review that produced this finding |
| Documents reviewed | Which of the five foundation documents were in scope |
| Finding | Plain-language description of what was found |
| Proposed action | What should be done about it at the next gate |
| Status | Staged / Incorporated / Resolved |

---

## Finding categories

| Category | Meaning | Routing |
|----------|---------|---------|
| Act now | Active risk identified — immediate attention required | Routes to advisor queue as Tier 1 decision; real-time High alert sent |
| Note for phase-gate | Non-urgent improvement — address at next advancement gate | Written to this file; automatically included in next gate review |
